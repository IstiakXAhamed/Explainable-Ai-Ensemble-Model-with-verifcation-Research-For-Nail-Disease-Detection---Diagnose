Nail Disease Detection via Hybrid Deep Learning & Ensemble Architecture
📌 Project Overview
This repository outlines the architecture and methodology for a diagnostic AI tool designed to classify various nail diseases from clinical images.

The project overcomes the limitations of classical machine learning on image data by utilizing a Hybrid AI Architecture. It leverages deep learning for robust spatial feature extraction and a classical machine learning ensemble for high-precision decision-making.

🔬 Target Classifications
The model is engineered to distinguish between healthy nails and specific conditions, including:

Acral Lentiginous Melanoma (ALM)

Onychogryphosis

Nail Clubbing

Nail Pitting

Blue Finger Syndrome

Healthy Nails

🧠 Architectural Design (The "Super-Hybrid" Model)
Instead of relying on a single end-to-end model, this system splits the diagnostic process into two distinct engines: The Vision Engine and The Logic Engine.

Phase 1: Deep Feature Extraction (The Vision Engine)
Backbone: Transfer Learning via MobileNetV2 (pre-trained on ImageNet).

Process: The top classification layers of the CNN are bypassed. Instead, the network acts purely as a sophisticated feature extractor, translating raw 2D image pixels into dense, 1,280-dimensional mathematical vectors.

Why: This prevents the loss of spatial hierarchies and texture relationships that occurs when flattening images for classical algorithms.

Phase 2: Triple-Engine Ensemble (The Logic Engine)
Input: The standardized 1D feature vectors generated by Phase 1.

Algorithms: A robust trio of classical classifiers:

Support Vector Classifier (SVC): Utilizing RBF kernels for complex boundary mapping.

Random Forest (RF): For handling non-linear feature interactions and preventing overfitting.

Logistic Regression (LR): For baseline linear probability calibration.

Decision Protocol: The final classification is determined using a Soft Voting Strategy, calculating the argmax of the sum of predicted probabilities from all three estimators to capture maximum nuance in borderline cases.

📊 Data Pipeline & Preprocessing
Scale: Trained and validated on a dataset of approximately 3,800 clinical images.

Augmentation: To prevent memorization and improve generalization, dynamic data augmentation was applied (rotation, width/height shifts, shearing, and zooming).

Validation: Strict 80/20 data splitting with generator shuffling disabled during extraction to guarantee absolute alignment between deep features and categorical labels.

🚀 Key Engineering Takeaways
Successfully merged deep learning feature spaces with classical ML decision boundaries.

Mitigated the "flattening problem" inherent in running image data directly through SVMs/Random Forests.

Optimized model efficiency, allowing the final classification engine to train instantly on compressed feature vectors rather than millions of raw pixels.
